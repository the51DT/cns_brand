Image Box Component

  • addClass 로 클래스를 이미지를 감싸고 있는 .img-box 에 이중클래스를 부여하여 width 조절할 수 있습니다.

  • PC, MO 이미지 url을 따로 넣을 수 있으나, PC 이미지 url 입력은 필수 조건입니다.

Image Box Component type 1. pc/mo 이미지가 같은 경우

<% const iamgeData2=[{ pc: 'https://placehold.co/1144x643', }] %> <%- include('src/component/cmp_image_box.html', Object.assign({}, locals, { jsonData : iamgeData2, alt : '유통 산업 성장의 Key 최대 효율을 당성하는 성공 DX', addClass: 'test2' }))%>

Image Box Component type 2. mo 이미지 비노출 처리의 경우

  • addClass:'img-box__only-pc'

  • pc/mo 이미지 미노출 처리 시 .cmp-wrap 에 기본적으로 붙는 margin-bottom 은 0 이 된다. (content.scss 참고)

<% const iamgeData4=[{ pc: 'https://placehold.co/1440x400/FF0000/FFFFFF/png', }] %> <%- include('src/component/cmp_image_box.html', Object.assign({}, locals, { jsonData : iamgeData4, alt : '유통 산업 성장의 Key 최대 효율을 당성하는 성공 DX', addClass: 'img-box__only-pc' }))%>

Image Box Component type 3. pc 이미지 비노출 처리의 경우

  • addClass:'img-box__only-mo'

  • pc/mo 이미지 미노출 처리 시 .cmp-wrap 에 기본적으로 붙는 margin-bottom 은 0 이 된다. (content.scss 참고)

<% const iamgeData3=[{ pc: 'https://placehold.co/1440x400/aecbe9/FFFFFF/png' , }] %> <%- include('src/component/cmp_image_box.html', Object.assign({}, locals, { jsonData : iamgeData3, alt : '유통 산업 성장의 Key 최대 효율을 당성하는 성공 DX' , addClass: 'img-box__only-mo' }))%>

Image Box Component type 3. pc/mo 이미지가 다른 경우
